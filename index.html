Define the roadway geometry the bridge follows: the horizontal alignment (the path in plan — tangents, curves, spirals) and the vertical profile (elevation along the route — grades and vertical curves). Everything else in the tool is positioned relative to this. Graphics at the bottom show the result.

Horizontal Alignment

Station NNN+NN.NN. Bearing as quadrant DMS (N17°48'37"E) or azimuth degrees. Northing→Y, Easting→X. Positive R curves right.

Vertical Profile

PVIs · first row is Start point, last is End point; interior rows are PVIs with vertical curves

Required inputs (solid boxes): station and elevation for each point, plus a vertical curve length for interior PVIs. Optional overrides (dashed, italic boxes): grade in/out are computed and shown, but you may type one to override it — editing grade-out holds that PVI and recomputes the next point's elevation; an overridden grade turns solid oxide. The Start and End rows are the profile endpoints (no curve). A.D., K, BVC/EVC and high/low point compute automatically; a vertical curve is centered on its PVI and spans BVC to EVC = its length.

Cross Section

global default

The PGL = baseline = reference line, fixed at offset 0 — the deck elevation equals the vertical profile elevation here. The crown (deck high point) and bridge CL are each entered as offsets from the PGL; enter 0 for either to place it on the PGL. Per-unit cross-slope overrides set in the table below.

Girders & Deck Edges

global girder offsets (ft from the Bridge CL)

Girders, deck edges, and curbs are all measured from the Bridge CL (the center of the bridge), so they're symmetric about it by default. The Bridge CL itself is positioned relative to the PGL by the Bridge CL offset field above — changing that slides the whole deck (edges, girders, curbs) together. Per-span edge overrides and per-support curb offsets are in the substructure table.

Superstructure Depths

pavement & top of deck → beam seat

The vertical profile and PGL are defined at top of pavement (top of HMA). The top-of-deck (structural surface) is the profile elevation minus the HMA thickness. Beam seats hang off top-of-deck, so they drop with it automatically.

Beam seat = top-of-deck − deck thickness − (min haunch + camber) − beam depth − bearing assembly. Camber is the positive midspan ordinate; it lowers the seat. All depths on this panel are in inches.

Substructure Units

Skew measured from alignment normal (0° = perpendicular). Dual bearing offset = distance back/ahead of unit station.

Per-span girder & deck-edge overrides

Most bridges use the same girders and deck width everywhere — if so, you can ignore this table entirely; every span uses the global values above. Use it only when a particular span differs (e.g. a span has a different number of girders, a flared deck, or a different edge offset). Each greyed cell shows the global value it would inherit; type a value to override just that span (it turns solid), or clear it to revert.

Validation · known points

Spot-check the model against points whose elevation you already know (e.g. from a plan or independent calc). Enter each point's station, transverse offset (from the PGL, offset 0), and expected top-of-deck elevation in its own box. Each is compared to the computed elevation and flagged if it differs by more than the tolerance.

Beam Seat Elevations · at bearing lines

Seat = top-of-deck − deck thickness − (min haunch + camber) − beam depth − bearing assembly. One row per bearing line; dual-bearing piers report back & ahead separately, each using its own span's camber/depth/bearing.
